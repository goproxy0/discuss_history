Reported by shuangzh...@gmail.com, Aug 24, 2014 
IP获取方法：
cmd中输入：nslookup ad.doubleclick.net
查询该域名的IP，结果一般都可用（如果IP开头为203.208的话那是谷歌北京的IP，如果不是，那恭喜你，你发现了一组可用于GAE的新国内IP）。目前还发现似乎谷歌北京的IP又可以代理GAE了（仅限于GAE）。大家可以试试。
另外要用这组地址来代理，需要对proxy.ini做一点更改（可以写在proxy.user.ini）
[profile/ipv4]
.google.com = withgae
.google.com.hk = withgae
.googleapis.com = withgae
.googleusercontent.com = withgae
.googletagservices.com = withgae
.googletagmanager.com = withgae
.gstatic.com = withgae
.ggpht.com = withgae
.googlegroups.com = withgae
.googlecode.com = withgae
.android.com = withgae

其实看得懂的都知道，就是把所有谷歌流量全部转为GAE代理.

working version for 3.1.25
proxy.user.ini

[gae]
cachesock = 0
headfirst = 1
keepalive = 0
obfuscate = 1
pagespeed = 1
validate = 1
transport = 0
options = 

[iplist]
google_cn = ad.doubleclick.net
google_hk = ad.doubleclick.net

[profile]
.google.com = withgae
.google.com.hk = withgae
.googleapis.com = withgae
.googleusercontent.com = withgae
.googletagservices.com = withgae
.googletagmanager.com = withgae
.gstatic.com = withgae
.ggpht.com = withgae
.googlegroups.com = withgae
.googlecode.com = withgae
.android.com = withgae
.google.cn = google_hk
.appspot.com = google_hk
.googleusercontent.com = google_hk
