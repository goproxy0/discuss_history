Reported by SiuFe...@gmail.com, Jun 8, 2014 
升级到最新版后（目前版本：v3.1.17-5），如果google drive （指win客户端）不能使用，并且客户端出现红字错误：accounts.google.com:443 tlsv1 alert unkown ca的话，可以尝试更改proxy.ini设置
[ipv4/http]
fakehttps = .google.com.hk|.googleapis.com|.googleusercontent.com|.youtube.com
nofakehttps = scholar.google.com|scholar.google.com.hk|.google.com

主要改动是将.google.com从fakehttps移到nofakehttps，改动后Google drive连接非常顺畅。

#16 z.jonath...@gmail.com 
综合整理下各位大师的方法：

====proxy.ini中====

[profile/ipv4]字段做如下添加：
accounts.google.com = google_hk,forcehttps,nofakehttps
clients3.google.com = google_hk,forcehttps,nofakehttps
drive.google.com = google_hk,forcehttps,nofakehttps
upload.drive.google.com = google_hk,forcehttps,nofakehttps
.google.com = google_hk,forcehttps,nofakehttps
googledrive.com = google_hk,forcehttps,nofakehttps

[gae]字段修改
sslversion = SSLv23

=====打开IE-Internet Explorer设置-高级-安全====
勾上SSL2.0，SSL3.0
去除TLS1.0，TLS1.1，TLS1.2

至此困扰多年的问题问题完美解决。

测试环境：Win7+IE8+GAE3.1.22+GDrive1.17.7290.4094
