最近youtube问题连连，goagent频繁更新，是不是更新更到了手软？？？

自己制作了一个批处理辅助自动更新程序：

默认第一次安装会自动从官网下载最新goagent，并启动自动配置向导，就算是没用过goagent的新手也可以轻松配置。

正常使用，启动配置批处理，按0即可实现一键升级更新，方便快捷，会自动读取之前配置过的appid并导入，无需再手工下载配置。

并带有其他更多功能，可自己探索。。。

全部用批处理实现，自解压包，懂批处理的可解压看到所有文件源码，安全可靠！

-----------------------------------------------------------------------

#26 daxiaop...@gmail.com 
按照楼主的修改（共两部分修改）：
1、
如果想提前自行替换的话，可以右键编辑桌面goagent快捷方式文件，找到下面一行（整个是一行，没有断行...）：
@path %PATH%;%windir%;%windir%\system32
@netstat -an|find "LISTENING"|find ":8087" && set HTTP_PROXY=http://127.0.0.1:8087 && set HTTPS_PROXY=http://127.0.0.1:8087
@wget.exe -O goagent.zip --no-check-certificate https://nodeload.github.com/goagent/goagent/legacy.zip/3.0 ||echo %downloaderror%  &&pause &&exit

替换成：

wget.exe -O goagent.zip http://git.oschina.net/goagent/goagent/repository/archive?ref=3.0 ||echo %downloaderror%  &&pause &&exit

2、
将如下三行字段：

for /f "delims=" %%a in ('dir /b /d goagent-*') do @set pathA=%%~nxa
xcopy "%pathA%"\*.* . /s /e /y 1>nul 2>nul
rd /s /q "%pathA%" 1>nul 2>nul

修改为：

xcopy goagent\*.* . /s /e /y 1>nul 2>nul
rd /s /q goagent 1>nul 2>nul

经测试完美成功！

导入的当然是新解压版本的证书了，为了达成通用包的目的，在官方如果更新CA.crt文件后，能够最大化的正常使用（如果自动升级后不主动导入证书，官方哪天升级更换证书文件了，会导致大量因自动升级而无法使用的情况），况且，命令导入证书是覆盖的，保证你每次更新用到的证书永远是最新的！

如果你自己调整过证书（删除CA.crt并重新生成该文件），可以在文件生成后，用主界面的5来重新导入即可！
