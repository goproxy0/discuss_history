Issue 7396: 终于找到看Youtube视频会断流的解决办法啦！

Status:  Done

Reported by ilovewei...@gmail.com, Nov 22, 2012 

原因一共有两个：
①你的Goagent版本不对！如果你是伸手党，那你以下教程教程不用看了，老实用着现在的版本，看Youtube视频虽然会断流，你只需要拉一下进度条就又可以看了。
②最大的原因是你连接到google服务器的IP地址被污染了！就相当于你平时google可以上，但是一段时间过了之后又连接不上，让你误以为google服务器有问题！
这就是现在为什么2.0版本之后为什么会有内置IP的功能。

=================解决方法===================================
①换回1.10.0版本，或者2.0版以前的版本都行！2.0版之后基本都是会断流的，因为有内置IP的功能。
②修改proxy.ini里面的信息：

[google_cn]
mode = https
hosts = 74.125.228.50

mode一定要改为https，不然连接不上！
hosts其实只需填一个IP地址就可以啦！IP地址自己找，估计我放出的这个IP地址在不久之后也会出现断流的情况！
放出这个IP地址是为了方便大家测试！虽说这个地址速度很慢，但稳定性还是有的，起码不会断流！
我自己好不容易找到一个快速而且不会断流的IP，不过总感觉迟早会被封，祈祷吧。
同时希望大家能找到适合自己的，大家最好还是不要公布出来，不然又很快挂掉。

==========以下是解决过程，有兴趣可以看一下====================

相信这里也有很多像我一样的菜鸟，在这一个月以来，饱受看Youtube会断流的痛苦！
在issue这里看到很多提问，一个说要开异步DNS，一个又说DNS受污染了，另一个说Https本来就是会断网的，但却没有一个真正满意的答案。
今天无意看到一个人问Goagent程序是不是内置IP了？突然觉悟，莫非是作者所说的随机IP。
于是用软件一看，还真是的，那个proxy.exe，居然连接了那么多不同的IP，而那些IP都不是我自己所找到的。
于是我去翻了以前的版本一个一个测试，发现果然是这内置的IP作怪！！！
经测试，2.0.0之后的Goagent都是设置有内置IP的，不过就一个版本例外，那就是2.0.13，而2.0之前版本的不会出现这种问题。
也就是说只要你用2.0版之前和2.0.13版就不会出现这种问题！这里我推荐用1.10.0版比较好，好像也就是1.0最后的版本！
