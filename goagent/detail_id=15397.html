#11 anesli...@gmail.com 
To #10 androids...@gmail.com
dnscrypt和dnsforwarder混合用确实会出现冲突，无论先运行那个，后一个都无法运行，这是因为这2个软件默认都是监听127.0.0.1 53端口，所以需要监听不同的地址和端口。
目前dnscrypt没有找到改监听地址和端口的地方。
我使用的pwx-dns-proxy可以修改监听地址和端口，可以两者配合用。pwx-dns-proxy监听127.0.0.2 53,dnscrypt监听127.0.0.1 53，这样两者就不会冲突了。
我没有使用过dnsforwarder，貌似dnsforwarder和pwx-dns-proxy的配置文件差不多，应该是可以修改的。

#19 uuu...@gmail.com 
其实完全可以用bind+dnscrypt+dnsforwarder，在bind9的配置文件named.conf.options中配置
fowarders{
127.0.0.1 port 5353 //dnsforwarder
127.0.0.1 port 5053 //dnscrypt
}
然后分别配置dnscrypt的端口为5053，dnsforwarder的端口为5353，对外部还是53端口的普通dns server
