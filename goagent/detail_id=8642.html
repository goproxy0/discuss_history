#2 cyrilcui...@gmail.com 
你的问题太多了  好吧我这会儿无聊 一个一个解答吧
localhost 对应 IP 地址 127.0.0.1 是 loopback 连接, 要调用网络协议层.
file:// 是直接读取本地文件, 不走网络.
第三个斜杠 / 代表 unix 文件系统的 root, 跟文件协议 file:// 无关.
总地来说, 现有的 iOS 客户端对 PAC 做的不是很好, 你不能按照程序里 "写好的" 方式来实现 PAC. 原因如下:
1. iOS 客户端太老旧, 自带 py 脚本现在早已不能使用.
2. 旧版脚本和新版对 PAC 的连接端口等方式定义不同, 若自行更新 py 脚本, 则需要修改连接方法.
3. 不能自动更新 PAC 这个别想了.
解决办法:
Step1. 更新 py 脚本, 方法: 把电脑版的 py 脚本改文件名之后替换上位机文件.
Step2. 修改 ini 配置文件, 注意要同步更新新版对应的配置文件, 因为内容有很大改动. 
Step3. 上传 pac 脚本, 方法: 用电脑运行客户端, 注意打开 pac 模块, 使用一次 pac 功能, 则脚本被自动更新为最新版本. 再把更新后的 pac 文件改文件名之后替换上位机文件.
Step4. 在 iOS settings 中使用 file url 指向更新后的 pac 文件. 注意路径一定要填对.
关于自动更新, 如果你是一般用途的话, 经常上的网站是不会有太多变动的, 被强的总会被强, 所以几个月不更新 pac 也没有什么问题.

3楼回答回复：
1、PAC文件我自动代理昨天删了重装后解决问题了，我觉得是IOS版的Goagent欠稳定还不知道怎么的。我觉得现在自动引用pac文件，还是得用url引用较好，用这个地址：http://127.0.0.1:8086/proxy.pac
同时这个功能需要在inc里配置，有这么一块内容：
[pac]
enable = 1                            //这个必须是1,开始pac自动引用功能
ip = 127.0.0.1                        //本机地址
port = 8086                           //引用端口
file = proxy.pac                      //引用文件
gfwlist = http://autoproxy-gfwlist.googlecode.com/svn/trunk/gfwlist.txt

如上配置的话，自动引用地址就是：http://127.0.0.1:8086/proxy.pac

试试这个功能有没有开启成功的最好办法就是打开网页浏览器safari，在地址输入http://127.0.0.1:8086/proxy.pac，如果能打开，让你下载说明这个服务开通成功，
之前我就这样试过打不开，所以我只用手动用端口成功，自动死活不成功。现在解决问题了。
自动有一点好处就是会自动判断，是本地的不过代理，是国外的好像自动用代理。
