为什么我只在反对公开ip的帖子回复？
首先，并不是每条这样的帖子我都回复，目前我也就回了几个言辞极为夸张，甚至到神经兮兮的人帖子 至于其他无关帖子 尤其是小白的水贴 我根本就没必要回

曾经负责gfw 的一个研究所每年招几十个博士？你一个个体能找到的IP 他们找不到？
解释一下已知 Google ip 段被封端口的原因，说了，最近是 5×5敏感时期，加大了封锁力度，不只是个Google 几乎所有境外的 网站 https 端口都受到了干扰，gfw具有足够的资源（几十台超级计算机和分布全国的骨干节点）甄别https证书，验证证书如果是 Google CRT 直接丢包，这是非常容易的事

关于 公开 IP 的担忧，我也表示理解，因为 gfw 有个基于流量分析的方法，开始某些 GCC 服务器知道的人少，流量不大，所以那些少数人用的很爽，前面有几位同学说的 gcc 服务器被干掉 或许确实是因为最近知道的人太多 流量太大 尤其是很多人用这些ip看高清油管，很快就让流量上到了gfw干预的阈值 所以 从这个角度说 我也是理解那些不要公开 ip 的同学 毕竟这样会大大增大流量 暴露 ip 关于香港 gcc 的一点提醒 香港 gcc 的ISP ban 了 ping 服务 如果你ping不通 可以直接用 ip 输入地址栏 看看是否连上 直接输入地址栏默认是 http 目前干扰不明显 但是 https 会很慢

不公开ip 让某些ip让少数人独享 当然是一个保护ip 的方法 但这不是 goagent 的初衷 如果 shadowsocks 的作者开始也这么想 一直把那个工具不公开 大家会有更多可选工具吗？隐藏ip是被动防御 主动进攻是公开更多 ip 让每个ip的流量相对降低 gfw一直没有敢直接ban掉所有google 当然 这方面不同人有不同的看法 有自己的选择 大家博弈吧

#33 wwbf...@gmail.com 
#23 每封锁 干扰一个ip 一个域名都是需要成本和服务器资源的,尤其是对整个ip的封锁成本很大.所以它不会检测所有的流量,并且完全封锁所有的"非法"ip.就现在而言,它连证书的Server_name都没有检测.所以只要你获得了合适的ip就可以通过https方式顺利连接各种''非法网站'.这是客观事实,gfw也的确没有这么做.
Google的ip十分不定,想要找到绝非易事,这无论是对你我还是gfw的工作人员都是一样的.我提供一个方法,你可以在消除了dns污染的情况下通过打开youtube视频试着去寻找*.googlevideo.com的域名,从而得到他的ip.它的域名和ip是一一对应的.一般在它的ip的附近都会存在证书为*.google.com的ip.如果它的速度不错.并且能ping不掉包的话,那就是一个好ip.当然,你可能会在打开到第100个视频或是200个的的时候得到了一个非74.125段或173.194段的ip.如果它的各项性能稳定的话,那你就成功了.或者你也可以全网搜索google的域名,一个一个去nslookup.可能会找到合适的ip.我这么说你应该能明白,这个工作量是巨大而且无趣的--有可能半天毫无所获.所以我希望大家能够更珍惜些,因为在你方便的背后都是别人巨大的工作量.
我知道你的初衷是好的,但这里的确不是普及网络知识的地方.但想要分析是否应该公开,应该公开到什么程度才能使得更多的人知道但又安全,为了达到这一点一些专业知识也是必要的.很明显,在这个时间点,在这里,这是不合适的.
如果我侵犯到了你,我表示真诚的歉意.但大家的初衷都是好的,我不希望因此吵起来,这就得不偿失了.
