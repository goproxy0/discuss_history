#1 zhao.fei...@gmail.com 
opendns的dnscrypt服务器用的是443端口，非传统的53端口。公司代理模式下可能会对端口或者协议过滤。如果连不上，可能是被你们公司的防火墙挡了。这种情况下没什么办法。其实goagent内置的dns解析也行，采用tcp查询+黑名单过滤，可以用了。如果在你们的公司的网络下可以使用公网服务器的dns解析（nslookup 测试一下就行），直接开启goagent的dns解析，然后修改本机dns配置就行了。如果你们公司连外网的dns解析都屏蔽了，那就需要将你们公司的dns服务器替换proxy.ini中dns配置。

#3 zhao.fei...@gmail.com 
这样吧，你进入命令行：
1. cd /d "opendns安装目录"
2. 执行dnscrypt-proxy.exe
如果看到[ERROR] Unable to retrieve server certificates，说明你的网络受限。
然后在命令行中执行：nslookup sina.cn 208.67.220.220，看有没有出现：
Name:    sina.cn
Address:  203.90.242.118
如果出现，表示你还可以使用外网的dns解析。可以更改proxy.ini配置开启dns解析，并将本机dns设置成127.0.0.1就行了。
如果不出现，你可以试下nslookup sina.cn 114.114.114.114，如果都不行。说明你不能使用外网dns服务器。需要使用你们公司自己的dns

#5 zhao.fei...@gmail.com 
把你们公司的DNS替换proxy.ini中的[dns]下server设置
如：
servers = 8.8.8.8|8.8.4.4|223.6.6.6|114.114.115.115
改成
servers = [公司DNS]

DNS解析应该就行了。也可以将dns解析打开供系统其他软件查询dns，
设置proxy.ini中[dns]enable = 1，同时设置系统的dns为127.0.0.1

#15 Zane...@gmail.com 
运行dnscrypt-proxy

将本机网卡dns设置为127.0.0.1

然后将proxy.ini里的DNS选项按照如下修改：

[dns]
enable = 0        <-----注意，这里不要开启，不然会和dnscrypt-proxy冲突
listen = 127.0.0.1:53
servers = 127.0.0.1
