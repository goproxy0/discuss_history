我想很多人也注意到了某个台湾段 IP 的陷落
至于为什么，我可以很明确的说，和某明文 hosts 有直接关系
基本上是这样的关系：
某 IP 添加到某 hosts 中，IP 挂，换 IP 添加，IP 再挂，再换

我们批判 hosts 么？没道理，错在 GFW 而不在他们
只是他们不够聪明，盲目的在 hosts 这个东西上追求速度而不在意可提供的 IP 库的大小
台湾那个段就剩为数不多的 IP 完全可以用别的速度稍逊但是有大量可用 IP 的段来替代换在 hosts 里

穷则独善其身，达则兼济天下

GoAgent 能媲美台湾某段的可用 IP 我手里还有两段，但是我不能分享给大家，不是我不愿意，只是我不希望我自己的路子越来越窄，当我找到了更多我会分享其中一个段给大家

当河水越来越湍急，当然是游泳好的先游到对面，如果游泳好的还要背着不会游泳的游，很可能带来的结果是全部被河水冲走

所以我只给大家一个提示，尝试着去找 GoGo Tester 之外的 IP 吧
如果找到了，穷则独善其身

http://goo.gl/I4213J
http://goo.gl/8X3xMJ

当找到多个好用 IP 段的时候，再放出一两个段，达则兼济天下

Re: #1 #2 #3

只是看到大家吐槽说某段 IP 挂了然后说大家可以放眼 gogo tester IP 库存之外而已
我并没有怪罪 hosts 分享者啊，但是我觉得 hosts 的最主要的用途在于「覆盖全」而不在于「速度快」，所以用还拥有大量 IP 的 IP 段比较好

我毫无恶意，不知怎么就被你们生造出了恶意，我只是不希望太多人因为 IP 挂了就放弃 GoAgent 或者说 GoAgent 不如以前好用了，因为 IP 还有很多，具体怎么找的是哪些我这明文写了 GFW 依然会看，说不定等我写完就挂了，曾经的 4000+ 那个段就是这个下场，一夜之间全死

我所言及的「聪明」是指能不仅给人翻墙的法子还能稍微的避免翻墙路子变的越来越窄，明文写于网上的 IP 自然是封的最快的，所以我不建议手上只有一两段 IP 的人就直接分享出来

我现在只找到了两段 GoGo 之外好用的 IP，所以暂时先是自用的，我找 IP 也没有任何人授我任何渔／鱼，完全都是自己摸索的，我仅仅是告诉大家使用 GoAgent 还是可以通过另外的途径找好用 IP 的一点，至于明白的人自然会去自己摸索，我要写明了方法，GFW 也会顺着路子干一样的事儿，现在 IP 越来越少，这样明文分享自然是不明智的

为什么 GoAgent 一直没有直接明文写 IP 的 IPlist 版本，大抵也是一样的道理，Phus.lu 在给大家提供翻墙方式的同时也会谨慎对待 GFW 的封锁，如果明文写 IPlist 肯定会被封的更快

至于搜索 GoGo 之外的方法我就提供一个：「挂港台日韩 VPN，Google 查询当地各地区不同 ISP 的 DNS（尤其是小服务商的），利用各种谷歌域名 nslookup」

然后我再重申一遍：错不在分享者，而在 GFW，没必要觉得我是秀优越，我只是觉得 Gogo 库存里最好的 IP 段死完了，大家可以放眼 GoGotester 之外罢了
