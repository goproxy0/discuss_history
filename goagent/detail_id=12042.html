Issue 12042: 用Centos 建立内网goagent代理服务器供网内其他设备使用

Reported by Zane...@gmail.com, Dec 23, 2013 
系统Centos 6.5 x86-64  minimal
按照顺序执行下面命令:

yum groupinstall "Development Tools"

yum install openssl openssl-devel pyOpenSSL libevent libevent-devel wget

cd
wget http://www.python.org/ftp/python/2.7.6/Python-2.7.6.tgz
tar xzvf Python-2.7.6.tgz
cd  Python-2.7.6
./configure --with-zlib=/usr/include
make
make install

cd
wget --no-check-certificate https://pypi.python.org/packages/source/p/pyOpenSSL/pyOpenSSL-0.13.1.tar.gz
tar xzvf pyOpenSSL-0.13.1.tar.gz
cd pyOpenSSL-0.13.1
python setup.py install

cd
yum install  pycrypto
wget https://ftp.dlitz.net/pub/dlitz/crypto/pycrypto/pycrypto-2.6.1.tar.gz
tar xzvf pycrypto-2.6.1.tar.gz
cd pycrypto-2.6.1
./configure
python setup.py install

cd
mkdir -p .pki/nssdb
certutil -d .pki/nssdb -N


wget -O goagent.zip --no-check-certificate http://goo.gl/qFyRk


unzip goagent.zip
mv goagent-goagent* goagent
cd goagent
cd local
vi proxy.user.ini

将如下内容写入proxy.user.ini
[listen]
ip = 0.0.0.0
visible = 0

[gae]
appid = yourappid
[pac]
ip = 0.0.0.0

[paas]
listen = 0.0.0.0:8088

[php]
listen = 0.0.0.0:8089

[dns]
listen = 0.0.0.0:53
-------------------------------------

保存后，用vi打开proxy.pac文件,将所有的127.0.0.1替换成你当前centos linux的ip地址（假设为192.168.1.2）
使用vi中的命令模式来完成批量替换：
:%s/127.0.0.1/192.168.10.2/g
:x

保存退出后，执行：
./proxy.sh


将goagent加入到自动启动:

vi /etc/rc.local
用vi打开rc.local，添加如下信息：

cd /root/goagent/local;./proxy.sh


完成后，将centos的SElinux关闭
vi /etc/sysconfig/selinux
将SELINUX=enforcing
修改为SELINUX=disabled

清空iptables：
/etc/init.d/iptables stop

然后就可以在其他PC、或手机设备上借助这一台linux进行翻墙了

首先在其他pc或手机上用浏览器访问如下地址(假设centosIP为192.168.1.2)：
http://192.168.1.2:8086/CA.crt

下载到证书后，安装即可（安装导入方式和本地goagent相同）

然后在Internet选项中--连接--局域网设置里，选择使用自动代理：
http://192.168.1.2:8086/proxy.pac

#9 xxnet....@gmail.com 
GoAgent不适合做公共代理，存在很多问题：
1. 证书需要每个机器导入，麻烦
2. Python性能不是特别高，用户多点就顶不住了
3. 稳定性：大量用户使用少量的google ip，很容易被GFW识别出来，封锁

对于公司资金和技术都足够去搭VPN了，这是更好的选择。
买个可以装OpenWrt的路由器，自己在上面装ShadowVPN或者OpenConnect作代理，会稳定很多，而且可以做到透明代理，客户端不需要作配置，只要连上就用
