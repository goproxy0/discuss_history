Issue 10728: 貌似不能看youtube 的问题解决了
4 people starred this issue and may be notified of changes. Back to list
Status:  New
Owner:  ----
Type-Defect
Priority-Medium


 
Reported by mz11ou1...@gmail.com, Nov 1, 2013 
我只看了一个视频，所以不能完全肯定问题解决了
1、下载最新版goagent并解压缩（不用下那个proxy.ini文件，新版goagent里面已经有这个文件了）
2、修改goagent配置，即修改proxy.ini，讲appid覆盖成自己的IDs
3、我goagent走的是IPV6通道
走https就可以了
在python.ini中你所选择的主机项的sites和forcehttps后面加上|.youtube.com
比如我选的profile = google_ipv6
-------------------------
[gae]
appid = yourIDs
password = 
path = /2
profile = google_ipv6
crlf = 1
validate = 0
-------------------------

就在[google_ipv6]的sites和forcehttps后面各添加了个|.youtube.com
---------------------------
[google_ipv6]
mode = http
hosts = 2404:6800:4005:c00::65|2404:6800:4005:c00::8a|2404:6800:4005:c00::67|2404:6800:4005:c00::2f|2404:6800:4005:c00::30|2404:6800:4005:c00::84
sites = .google.com|.googleusercontent.com|.googleapis.com|.google-analytics.com|.googlecode.com|.google.com.hk|.googlegroups.com|www.youtube.com/metadata_ajax|www.youtube.com/api/tag_suggest|www.youtube.com/api/upload_feedback|www.youtube.com/upload/|upload.youtube.com|www.youtube.com/gen|www.youtube.com/my_videos_upload|.youtube.com
forcehttps = groups.google.com|code.google.com|mail.google.com|docs.google.com|profiles.google.com|developer.android.com|.youtube.com
withgae = 
withdns = 
---------------------------
4、先关闭goagent软件（这一步别忘了），再清理全部cookies、最后关闭所有网页
5、重新打开goagetn软件，再打开youtube网页就可以看视频了
