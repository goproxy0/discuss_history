Issue 17571: 卧槽，GFW针对IPv6也DNS污染了
3 people starred this issue and may be notified of changes. Back to list
Status:  New
Owner:  ----
Type-Defect
Priority-Medium


 
Reported by zzq1...@gmail.com, Sep 28, 2014 
感谢 Issue 17527 提出的IPv6 DNS污染问题（看起来是最近发生的）。
本人在墙外，且有IPv6环境，于是做了一些测试，发现了如下的有趣情况：
1. 和IPv4一样，此污染双向工作（墙内到墙外污染，墙外到墙内也污染）
2. 和IPv4一样，不一定非要是有效的DNS服务器（我用的墙内IPv6地址： 2001:250:200:: 做的测试）
3. 和IPv4不一样，DNS记录的TTL值只有两种：300秒和25600秒（IPv4是300秒到几万）
4. 和IPv4不一样，A记录返回如下IPv4地址：
1.1.1.1
10.10.10.10
20.20.20.20

AAAA记录返回如下IPv6地址：
2003:ff:1:2:3:4:5fff:6
2003:ff:1:2:3:4:5fff:7
2003:ff:1:2:3:4:5fff:8
2003:ff:1:2:3:4:5fff:9
2003:ff:1:2:3:4:5fff:10
2003:ff:1:2:3:4:5fff:11
2003:ff:1:2:3:4:5fff:12
（信不信，这些应该是以2003:ff:1:2:3:4:5fff:开头的地址，最后一个数字随机）
10::2222
21:2::2
101::1234
2001::212
2001:da8:112::21ae
2123::3e12

5. 和IPv4不一样，此污染针对的域名如下：（我就发现3个）
包含"youtube.com"的域名
包含"facebook.com"的域名
包含"twitter.com"的域名

6. 和IPv4相似但又有点区别，GFW的伪造DNS包是粗制滥造的
IPv4时代：无论查询什么记录，均返回A记录
IPv6时代：A记录和AAAA记录返回伪造地址，其他记录（比如CNAME，NS，甚至ANY）居然返回了这种东西：

a. (root)  ??? unknown type 0 ???
b. www.youtube.com ??? unknown type 255 ???
c. *** Error: record size incorrect (1 != 49168)
d. www.youtube.com canonical name = (root)
e. www.youtube.com nameserver = (root)
最后都是*** UnKnown can't find www.youtube.com: Unspecified error

SOA记录返回了：
www.youtube.com
        primary name server = (root)
        responsible mail addr = (root)
        serial  = 6553600
        refresh = 46140416 (534 days 46 mins 56 secs)
        （GFW已经疯了）
        retry   = 0 (0 secs)
        expire  = 0 (0 secs)
        default TTL = 0 (0 secs)

*** Error: record size incorrect (25 != 49168)

Non-authoritative answer:
(root)  ??? unknown type 0 ???

充分证明GFW工程师是SB

对了，本人在墙外测试有两个限制：（谁叫墙内没有IPv6的DNS服务器）
1. 无法测试GFW是否针对TCP的查询进行阻断
2. 无法测试GFW是否针对UDP非53端口的查询进行丢弃或污染
有IPv6环境的网友们可以帮我测一下，并贴出结果。在此表示感谢
测试方法：

nslookup -vc dl.dropbox.com 2001:4860:4860::8888
针对限制2: （不能用nslookup，要用dig）
dig -p 5353 @2620:0:ccc::2 www.youtube.com
