Reported by df...@dfeng.tk, Jun 17, 2012 
我新装了 archlinux 系统，只装了 xfce4 桌面，python2 环境，全是默认设置。
运行命令：python2 /path/to/goagent/proxy.py 能正常运行。firefox 浏览器导入证书后，浏览一些https网站提示证书安全问题，比如 arch 的官网wiki：https://wiki.archlinux.org。而浏览 certs 目录下有证书的网站则没有问题，比如：https://plus.google.com。
估计是因为缺少一些 python 组件包，不能在 certs 目录下生成证书造成的。
我搜索了issues，在这个issues：https://code.google.com/p/goagent/issues/detail?id=2593 找到解决办法：安装 python2-pyopenssl。安装后能正常浏览 https 网站了，比如上面的 wiki 网站，初次浏览能在 certs 目录下生成两个证书。
可是有一个问题没有解决，就是上面那个 issues 链接里提到的第一个问题：删除 proxy.py 所在目录下的两个证书 CA.crt CA.key 后，运行 proxy.py 出错，不能重新生成这两个证书（在 windows 下是可以的），错误信息如下

Traceback (most recent call last):
  File "/path/to/goagent/proxy.py", line 1298, in <module>
    main()
  File "/path/to/goagent/proxy.py", line 1275, in main
    CertUtil.checkCA()
  File "/path/to/goagent/proxy.py", line 525, in checkCA
    [os.remove(os.path.join('certs', x)) for x in os.listdir('certs')]
OSError: [Errno 2] No such file or directory: 'certs'

是因为还缺少某些组件包吗？求解，盼复。

我也遇到了这个错，但是我已经安装了pyOpenSSL，解决方法如下：

OSError: [Errno 2] No such file or directory: 'certs'

看报错，没有'certs'这个目录，在./goagent/local/目录下创建即可。
