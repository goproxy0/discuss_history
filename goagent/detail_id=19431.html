#20 shuangzh...@gmail.com 
系统有没有开启6to4只是一条命令的事情,和正版盗版无关
netsh int 6to4 set state en
即可开启,XP默认是没有IPV6支持,需要手动安装IPV6协议ipv6 install
想使用6to4必须要求电脑或者路由器具备一个公网IP,在内网使用192.168.*.*的地址是不能使用到6to4,和ISATAP一样.
2002:c058:6301网关地址实际上是由IPV4地址经过16进制转换得来,c0 58 63 01转换为10进制就是192.88.99.1,同样你的IP里面79e5:305e就可以分析出你的IP是121.229.48.94
另外6to4使用的中继Relay服务器为微软的192.88.99.1,国内也有一个大学的6to4服务器,可以通过netsh int 6to4 set relay *.*.*.*来设置,不过速度基本在400ms以上,没什么实际用处
电信原生IPV6地址开头为2401/2402/240e,部分地区已经开始有,不过大部分地区无法连接到国际互联网,甚至连腾讯的IPV6服务器也无法连接,基本就是个摆设.

这有什么可奇怪的。我问你在ipconfig/all中是否看到2002地址，是再正常不过的问题了。但你总是回避这个问题，还弄出个“网络连接器”。本人是XP，也不使用什么附加路由之类的部件。最通常的方法就是查看一下ipconfig。由于win7~8有“优先级别”的功能，或许有选择6to4优先的命令行，或者在组策略中可进行选择。而XP的ipv6就只是6to4功能，所以才有XP对ipv6的支持不够之说。反过来是否也可以认为，win7~8对6to4的支持没有XP好呢？ 
你无意中提到【goagent开启时也有信息，如果没有会自动用ipv4】，显然，你的goagent设置是ipv6= 0，如果你的ISP还没有ipv6服务，你可以尝试将goagent设置为 ipv6= 1 (当然要配合ipv6的IP)。如果你的goagent不能够访问网站，说明你的6to4没有开通，你当然在ipconfig看不到2002的地址!!!

联通的Teredo隧道快吗？
使用国外的r20---sn-hp576m7s.googlevideo.com是可以访问youtube的，也是可以看视频的，但是由于Teredo隧道的优先级问题，系统会优先解析ipv4地址，所以只能使用ipv6 hosts，以前的ipv6 hosts是可以用的，但是后来google把youtube视频服务器的地址从*.c.youtube.com格式的地址变更为*.googlevideo.com格式的地址后，视频服务的地址变得非常多，hosts就失效了，我曾经尝试收集所有的地址，但是发现地址太多了，似乎是不现实的。所以改变了策略，提升Teredo隧道的优先级，通过提升优先级可以实现双栈条件下优先解析ipv6，youtube就又可以访问了。
你用的是联通的服务器，不知道联通会不会劫持对youtube的访问？
