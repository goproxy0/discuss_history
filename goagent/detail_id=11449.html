Issue 11449: Goagent 官方永不升级 1.8.0 版，搞定一切 403、404、reset 问题

Reported by westmin...@gmail.com, Nov 13, 2013 
为何要用永不升级 1.8.0 版？小白问题不回答，谁用谁知道。（简单就是最好的）

1.官方下载地址 goagent-1.8.0.zip：https://github.com/goagent/goagent/archive/v1.8.0.zip

2.上传服务端

1) 如果想保留 3.0.8 版的服务端
解决方法：修改 1.8.0  版 server\python\app.yaml 文件中的 version:1，比如改成 version:180

2) 最好为自己的appid加密，防止别人盗用
解决方法：https://code.google.com/p/goagent/wiki/FAQ 第 27 点

3) 因为 1.8.0 版的 uploader 已过时，上传会报 403 错误
解决方法：复制 3.0.8 版 server 目录下的 uploader.bat、uploader.zip 到 1.8.0 版 server 目录下，替换掉原有的

3.配置客户端

1) 修改 local\proxy.ini 文件 （只列出需要修改的地方）
[listen]
visible = 0 （不在任务栏显示，只在通知栏显示）
[gae]
appid = 你的appid（如果如前面所说修改了version，应该这样填：version.你的appdi，比如我的是这样的：180.fxckgfwlaomu0|180.fxckgfwlaomu1，多个id用 “|” 隔开）
password = 你在服务端设置的密码（如果如前面所说添加了password，在这里填，注意是否大小写和中英文状态）
profile = google_hk（我这里 google_cn 已经连不上了）
[proxy] （需要先通过公司代理或小区代理或校园代理的内网用户，在这里设置前置代理）
[google_hk]
mode = https（我这里 http 方式已经连不上了，相信全国都是这样）
hosts = www.google.com|mail.google.com|www.google.com.hk（用原本自带的IP，我这里已经连不上了，改成 3.0.8 版的设置）
[pac]
enable = 1 （使用 goagent 自带的 pac，告别一切耗资源有风险的浏览器插件，如 SwitchySharp、FoxyProxy）
port = 8086（1.8.0 版是 8089 端口，为保持一致，改成 3.0.8 版的 8086 端口）
file = proxy.pac（1.8.0 版是 goagent.pac 文件，为保持一致，改成 3.0.8 版的 proxy.pac 文件）
update = 1（我不知道这个选项什么作用，可能是判断是否要保持更新）
remote = （改成 3.0.8 版的地址的话，不会自动更新，因为 3.0.8 版的地址下载来是 txt 文件，能自动转换成 pac 文件，1.8.0 版没有这个功能）
[love]
enable = 1（有没有爱心就看你的了）

2) 如果如前面所说修改了 file=proxy.pac，请将 local\goagent.pac 文件重命名为 proxy.pac

3) 以管理员身份运行一遍 goagent.exe，会自动重新导入证书，如果不行，请手动重新导入
（https://code.google.com/p/goagent/wiki/FAQ 第 21、22 点）

4) 设置浏览器代理：https://code.google.com/p/goagent/wiki/InstallGuide#附：浏览器设置方法
