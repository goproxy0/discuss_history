Issue 3506: ipad 上的pac逻辑混乱？还是有bug？
2 people starred this issue and may be notified of changes. Back to list
Status:  Done
Owner:  ----
Closed:  Dec 2012
Type-Defect
Priority-Medium


 
Reported by jus...@gmail.com, Apr 29, 2012 
现象之一如下：

1，用苹果的twitter app，开sbs toggle goagent设置【自动代理】，用goagent的pac文件，走ipad 本身的local代理。

或者设置【手动代理】用局域网内另外一台pc上装的goagent 开的8087端口走pc上goagent，这些方式，twitter app都会报错，“加载推文失败（-1012）”

2，用上述环境测试另外一款twitter app -- tweetbot ，这些都能正常使用。

twitter app是刚升级的新款客户端，是否不支持代理？

现象之二：

ipad上用“空气污染指数” app，经常出现没数据更新，要在ipad代理设置里完全关掉代理才能刷新数据。


请问到底写到ios网络代理里面那个file://localhost/User/goagent-local/autoproxy.pac 

文件是怎么和sbs toggle配合的呢？不止这几个app，还有各种稀奇古怪的访问错误。


比如现象3：我设置了所有代理都走8087端口的pac文件如下：


        function FindProxyForURL(url, host)
{
            return 'PROXY 127.0.0.1:8087';
}


按说这样web就全部走代理了吧？可是经常发现，用浏览器访问 123cha.com 或者 ip138.com 得到的我上网地址并不是google的地址。

这些乱套的事儿，真是好乱啊。

相反，现在用pc上的goagent还没什么问题，pc通过本身的8087代理，twitter都能打开，u2b也没问题，只有ios上的行为一团乱糟糟，另外，如果将ios上的代理设置为ssh转的http代理端口走ssh出国，这些问题都没了。

如果用ssh转的http代理，
1，twitter app访问正常，tweetbot也正常
2，123cha.com 和ip138.com 看到的ip都是代理ip，都正常。

为啥ios上这个行为这么混乱呢？？
