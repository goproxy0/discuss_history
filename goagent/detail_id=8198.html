解决OpenWRT上无法运行新版GoAgent的问题
因为一直以来，我是在OpenWRT上搭Tor，所以没有真正关心过GoAgent，前两天想看YouTube，但觉得Tor看YouTube实在太慢，想起了GoAgent，于是下载了最新版的GoAgent2.1.11(40)，服务器端很顺利，PC客户端也没问题，但是OpenWRT上却出问题，主要是三点：1、缺少gevent，2、python运行退出，3、OpenWRT出现异常。经过两天捣腾，终于将上述问题解决，不敢独享，拿出来与大家交流，希望不足的地方能得到高手的指点。
过程如下述：
1、下载GoAgent2.1.11并解压，配置服务器端（过程省略，自己看官方教程）；将解压后local文件夹内的python27.zip内的gevent文件夹解压出来单独保存好（里面有30个文件）；删除local文件夹内除certs、CA.crt、CA.key、proxy.ini、proxy.py外的所有文件（网上有些说要改proxy.py文件，但事实是不用改的，因为proxy.py会自己寻找正确的路径），将local文件夹改名为goagent。
2、修改proxy.ini文件内监听的IP地址为OpenWRT的内网地址（我的是改为192.168.1.1）。
3、在goagent文件夹中新建两个文件，分别取名goagent（启动GOA）和goakill（干掉GOA），两个文件都要给予执行属性，也就是属性0755，下面是这两个文件的内容：
# goagent文件内容（这段shell是我在goagent官网上看到的，但是一些关键的地方是错的，容易给人误导，我将它修改过，现在没有问题）：
CheckProcess() {
	if [ "$1" = "" ];
	then
		return 0
	fi

	PROCESS_NUM=`ps -ef | grep "$1" | grep -v "grep" | wc -l`
	if [ $PROCESS_NUM -eq 1 ];
	then
		return 1
	else
		return 0
	fi
	}

while [ 1 ] ; do
	CheckProcess "python"
	if [ $? -eq 1 ];
	then
		sleep 3s
	else
		python /var/goagent/proxy.py
	fi
	done

# goakill文件内容：
kill -9 $(ps ax|grep 'goagent'|grep -v 'grep'|awk '{print$1}')
4、利用WinSCP上传goagent文件夹到OpenWRT的/etc目录中，也就是/etc/goagent。然后上传gevent文件夹到/usr/lib/python2.7目录中，也就是/usr/lib/python2.7/gevent。
5、在/etc/rc.local文件中（在exit命令前面）加入下面三行：
cp -a -f /etc/goagent /var
cp -a -f /etc/goagent/goagent /usr/sbin
cp -a -f /etc/goagent/goakill /usr/sbin
6、重启OpenWRT路由，用web登录上去，在busybox（webshell）中键入goagent启动goagent；做好OpenWRT的端口映射，现在你到那里都可以享受GoAgent的乐趣了。
需要注意的地方：
1、只能在webshell中启动goagent！！！不要尝试使用SSH启动goagent，更不要在OpenWRT的自动启动中启动goagent。
2、我用的goagent版本是2.1.11(40)，启动时能会显示没有安装gevent（因为我没法在OpenWRT中编译gevent），不过使用是没有问题了。如果是goagent2.1.11(49)以后的版本，goagent还需要有goagent.pem，或者可以自己用现成的crt文件update-ca-certificates一个，但是我的OpenWRT无法编译，我的linux主机也down掉了。但是除了多一行报错外，goagent还是可以正常运行的。希望goagent的作者可以给出生成好的goagent.pem。
3、OpenWRT要有足够的flash空间，或者外挂u盘。
4、如果在家外通过OpenWRT翻，你的信息会泄露，最好在goagent的基础上配置stunnel。
最后，感谢goagent的作者和所有付出的人。希望goagent能越来越好，能更稳定跨平台也能像tor一样稳定运行。另外，IOS版（0.41）的，我装上去就无法正常运行，不过元旦后我也比较忙，希望迟点有时间看看，到时和大家交流……
