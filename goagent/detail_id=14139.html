怎么说呢。。。
确实被干扰了，但是重点还是要在自己会配置。。
我大概说一下我自己的配置吧：

首先，改监听端口号：
[listen]
ip = 127.0.0.1
port = 9090
username =
password =
visible = 1
debuginfo = 0

其次，筛选这里用IP，不要用域名，而且只留一个HK和一个筛选出来的IP就行了：
[iplist]
google_hk = *.*.*.*（IP还是你们自己找吧，我只能说，用亚洲的服务器IP就好。）

再次，在域名中全都指定好，不要留空：
[ipv4/hosts]
mtalk.google.com = google_hk
talk.google.com = google_hk
talk.l.google.com = google_hk
talkx.l.google.com = google_hk
……………………（下面的全都改，太长了，不发了。。）

再次，IPV6清空如下。Pac关闭：
[ipv6/hosts]

[ipv6/http]
dns =
crlfsites =
withgae =
forcehttps =
noforcehttps =
fakehttps =
nofakehttps =

[pac]
enable = 0

还有，这里需要线程改6条，并且多加一个零给每个参数：
[autorange]
threads = 6
maxsize = 10485760
waitsize = 5242880
bufsize = 81920


经过以上修改……
陕西电信8MB，Youtube看1080P不需要暂停缓冲，直接播放可以流畅……
是很流畅的那种…………

嗯，Enjoy~！
