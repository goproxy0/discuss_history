Reported by yizhi...@gmail.com, Jan 9, 2013 
局域网A机，西数的my book live，debian系统，搭建有apache、dnsmasq服务，python 2.7.3 ；固定ip 10.0.1.2；goagent搭建于此

局域网B机，ipad等，尝试1:手动设置代理为10.0.1.2:8087 尝试2:设置自动代理，地址http://10.0.1.2:8086/proxy.pac

版本：goagent-goagent-v2.1.11-69-g48f1aeb

proxy.ini内容：

[listen]
ip = 0.0.0.0
port = 8087
visible = 1
debuginfo = 0

[pac]
enable = 1
ip = 127.0.0.1
port = 8086
file = proxy.pac
gfwlist = http://autoproxy-gfwlist.googlecode.com/svn/trunk/gfwlist.txt



问题：

当采用尝试1:手动配置代理服务器时，一旦在ipad上打开需要代理的网站，如facebook.com，通过控制后台发现代理在无限递增接口连接，请看附件截图；
另外，证书不能成功生成：
cp: cannot create regular file `/usr/local/share/ca-certificates//shares/Public/code/goagent/local/CA.crt': No such file or directory

当采用尝试2:设置自动代理时，没有任何代理效果，控制后台也无日志输出，浏览器直接输入http://10.0.1.2:8086/proxy.pac 也无法获取文件


请问我改如何解决这个问题？
