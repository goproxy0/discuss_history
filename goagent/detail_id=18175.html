Issue 18175: WIKI：解释一下证书更新和更新的我已知问题，欢迎补充…

Reported by yue...@gmail.com, Oct 22, 2014 
首先，更新证书起源于内置证书，内置证书方便所谓的小白使用，可是不安全，类似的issue有很多。

之所以一直保留了内置的证书，最近才更新是因为，最近频繁出现的“中间人攻击”，删掉默认的证书，goagent默认会创建一个独一无二的证书，其实在很早的时候有很多人已经使用了新生成的证书。

证书问题的解决，分两步：删掉旧证书，导入新证书。
删证书和导入证书又分为操作系统级，和浏览器级。（如今很多浏览器支持，浏览器导入证书，系统不需要导入就可以使用。）

能到这儿的，没有小白与菜鸟只说，只有愿不愿意去找的区别。

火狐的例子： 编辑--首选项--高级--证书
查看证书，找到goagent删除，再导入新的。

重启动goagent后会自己生成，可以强制删除下，重启goagent，确保生成证书，新生成的证书，以后依然可用。  最后这句，重要！！！

附在这里，总结一下我的做法,从自己的wikidpad上copy过来

            1)File System: Delete CA.crt, cacert.pem(private key), certs/
            2)certmgr.msc: Delete the system CRT
            3)Browser: a)Delete Cert b)Delete File: cert8.db c)Delte Cache
            4)proxy.user.ini: validate = 1
            5)Run GoAgent. New CRT genetated
            6)Browser: Import CA.crt
            7)certmgr.msc: Import the system CRT
