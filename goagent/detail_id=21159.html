Issue 21159: 在vps上用apache2不熟goagent的php代理（包括index.py的python代理）
5 people starred this issue and may be notified of changes. Back to list
Status:  New
Owner:  ----
Type-Defect
Priority-Medium


 
Reported by yqh1...@gmail.com, May 28, 2015 
goagent server的php服务自带四个文件，分别是index.php,index.py,index.js,relay.php。分别为php代理，python代理，nodejs脚本代理，php中继代理。它们用来自己在vps服务器上部署代理给goagent客户端用。
其中比较好用的是index.php和index.py

以下以centos为例，debian的方法类似。
1.php代理（index.php)
php代理部署最简单，稍微检索下就能出来一堆文章,如果想直接看index.py的部署直接跳到 2.
可以检索到很多种apache+php的组合:最简单的（mod_php5，直接处理php），mod_fcgid + php5-cgi，mod_proxy_fcgi + mod_proxy + php-fpm；
从这里可以找到多种php安装方法，同时也有安装wsgi的方法
https://www.linode.com/docs/websites/apache

选用最简单的mod_php，只要yum install httpd24 php56即可,注意centos6默认php是5.2，apache为2.2，我都换成高版本

然后在你的虚拟主机配置文件加上+ExecCGI即可
-----------------------------
# for php
    <Directory "/var/www/html">
        Options +ExecCGI
        Require all granted
    </Directory>
------------------------------

服务器设置好apache2和php启用后，需要增大php上传限制：
编辑 /etc/php.ini （debian下是 /etc/php5/xxx/php.ini) 修改以下值
-----------------------------------------------------------
session.gc_maxlifetime 10800
max_input_time         10800
max_execution_time     10800
upload_max_filesize    110M
post_max_size          120M
-----------------------------------------------------------

默认设置下，www主页的主目录在/var/www/html
为php代理增加密码，修改goagent的index.php，修改第二项$__password__ = '你的密码';
然后把goagent的index.php放到/var/www/html/proxy/下
然后修改本地goagent的proxy.ini中[php]段改变如下值，我一般习惯于在proxy.user.ini里修改
--------------
enable = 1
password = your passwd
fetchserver = http://xx.xx.xx.xx/proxy/index.php
--------------
再浏览器里设置代理：127.0.0.1：8088即可

2.python （index.py，默认情况下性能更好，但看youtube耗内存也不小）
利用apache的python模块wsgi，lighttpd不支持。
centos6默认用python2.6，先用update-alternatives --config python来改变默认python的版本为2.7

安装wsgi模块
yum install mod24_wsgi-python27 (debian: apt-get install libapache2-mod-wsgi)
同样在你的apache虚拟主机配置文件加上以下设置：如/etc/httpd/conf.d/VirtualHost0.conf
----------------------
# for python
<IfModule mod_wsgi.c>
WSGIScriptAlias /proxy/ /web/www/html/proxy/
<Directory "/www/html/proxy">
Options +ExecCGI
AddHandler wsgi-script .py
Require all granted
</Directory>
</IfModule>
----------------------
or
在 .htaccess 中， 添加：
<Files index.py>
    SetHandler wsgi-script
    Options ExecCGI FollowSymLinks
</Files>

最后，同上述上传index.php一样，增加密码后上传index.py到/var/www/html/proxy/下，
并且赋予可执行权限：chmod +x /var/www/html/proxy/indexl.py

然后修改本地goagent的proxy.ini中[php]段改变如下值
--------------
enable = 1
password = your passwd
fetchserver = http://xx.xx.xx.xx/proxy/index.py
--------------
再浏览器里设置代理：127.0.0.1：8088即可

3.index.js
这是nodejs脚本，从https://nodejs.org/download/下载node的二进制程序即可node index.js运行。运行后会自动监听8080端口。
我猜应该是在apache里用反向代理方式提供操作。因为nodejs耗内存也不小，就懒得去试验了。

4.relay.php 是中继代理，即在国外不熟php代理后，在国内的vps里用relay.php连接国外代理做中继。
方法略。

2.python （index.py，默认情况下性能更好，但看youtube耗内存也不小）

js不建议，太耗内存了。部署也简单，
下载node的服务端，然后运行 node index.js ，会告诉你在端口8080监听。
在apache2上启用proxy模块，设置反向代理。就一句话：
---------------------------------------------
ProxyPass /nodejs/ http://127.0.0.1:9001/
--------------------------------------------
然后你访问 http://xx.xx.xx.xx/nodejs/ 注意最后的斜杠，就会跳转到google页面。说明部署成功。
