我比较了一下能够导入的证书，可能主要差别在x509的扩展上。

能够导入的证书都有下面的信息
Basic Constraints
Certificate Authority:	Yes
Max Path Length:	Unlimited
Critical:	No

即x509 v3的扩展，我猜测Android 4.0系统对证书有要求，我做了一下自签名证书导入的测试。

调用openssl时指定参数如下
openssl req -new -x509 -extensions v3_ca -keyout ca.crt -out ca.key -days 3650

openssl.cnf配置中
[req] section 增加或反注释此行
x509_extensions = v3_ca

生成证书后，可以用命令
openssl x509 -in ca.crt -text -noout
查看含有v3 extension信息。

加密位数4096/1024都可以导入Android 4.0系统。

请看一下，如果可能，请发布一下x509 v3格式的证书，或者是否可能由自签名证书替换GoAgent自带证书。

#3 max.c...@gmail.com 
1. openssl 语句中的顺序反了，应该是：

openssl req -new -x509 -extensions v3_ca -keyout ca.key -out ca.crt -days 3650 -config ./openssl.cnf

2. PEM 密码可以从 KEY 中删除，方法是：

openssl rsa -in ca.key -out new.ca.key

#10 henryo...@gmail.com 
解决这个问题不需要重新生成证书和私钥。可以直接用GoAgent的私钥来重新颁发一个带v3扩展的证书，替换掉原来的证书即可。

方法：
1. 从CA.crt里头分离出私钥CA.key (这只是个文本文件，私钥就是其中第二段的内容，把文件单独存起来）
2. openssl req -key CA.key -new -days 3650 -nodes -x509 -extensions v3_ca -out new.CA.crt -subj '/OU=GoAgent Root/O=GoAgent/CN=GoAgent CA/ST=Internet/C=CN/L=Cernet' -set_serial 0
3. cat new.CA.crt CA.key > CA.crt 替换掉原来local目录下的CA.crt

new.CA.crt可以传到Android手机的sdcard上，然后用系统security下面install certificate导入。导入后即可正常连接https的网站，不提示证书问题。
