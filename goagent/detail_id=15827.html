Issue 15827: goagent已经监听127.0.0.1:8087，但是chrome连接不上，netstat全部是SYN_SENT。
4 people starred this issue and may be notified of changes. Back to list
Status:  New
Owner:  ----
Type-Defect
Priority-Medium


 
Reported by LiangJia...@gmail.com, Aug 15, 2014 
我的机器上是第一次打开goagent.exe没问题。关掉之后再打开，必然提示“SystemError：<libev> error creating signal/async pipe:No error”。

想调试，看源代码，就是停留在proxylib.py的get_listen_ip中，connect的时候提示上述错误。
    sock.connect(('8.8.8.8', 53))

connect是库函数，不知道怎么再调试下去了。也不知道这个libev库到底是怎么回事。

本来想，我打开之后不要关闭。但是才用了几分钟，就连不上，看8087端口，全是SYN_SENT。一共也就4、5个SYN_SENT，没有其他，不可能是连接数太多了。goagent窗口没有任何提示（除去之前正常使用的那些日志）。
  TCP    127.0.0.1:8087         0.0.0.0:0              LISTENING       4956
  TCP    127.0.0.1:51761        127.0.0.1:8087         SYN_SENT        2820
  TCP    127.0.0.1:51762        127.0.0.1:8087         SYN_SENT        2820
  TCP    127.0.0.1:51763        127.0.0.1:8087         SYN_SENT        2820
  TCP    127.0.0.1:51764        127.0.0.1:8087         SYN_SENT        2820

换用goagent-uv.exe，不会再提示上述的SystemError。但是还是经常连不上，8087端口，全是SYN_SENT。

因为127.0.0.1不好用wireshark抓包，就修改proxy.ini和proxy.pac，把ip改成192.168.1.4。再打开，就好了。
proxy.ini
    ip = 192.168.1.4
proxy.pac
    var autoproxy = 'PROXY 192.168.1.4:8087';

我再多试几次，看看是否稳定。但是有个麻烦，因为192.168.1.4是DHCP获取的，每次开机可能会变。这样总是要改proxy.ini和proxy.pac就太麻烦了，proxy.pac修改后还要重新导入SwitchySharp。
