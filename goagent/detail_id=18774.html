#2 Monistra...@gmail.com 
I checked, it is python's problem in ubuntu, I try to recompile python to solved this:

#!/bin/bash

sudo -s

mkdir instal_python_build
cd instal_python_build

# install some packages
sudo apt-get install -y zlib1g-dev dpkg-dev build-essential \
    libssl-dev python-dev libffi-dev libnss3-tools g++

# compile python
wget https://www.python.org/ftp/python/2.7.8/Python-2.7.8.tgz
tar xf Python-2.7.8.tgz
cd Python-2.7.8/
./configure --prefix=/opt/python2.7.8 --with-threads
make -j8
make install

# install setuptools
wget https://pypi.python.org/packages/source/s/setuptools/setuptools-7.0.tar.gz
tar xf setuptools-7.0.tar.gz
cd setuptools-7.0/
/opt/python2.7.8/bin/python setup.py install

# install pip
wget https://pypi.python.org/packages/source/p/pip/pip-1.5.6.tar.gz
tar xf pip-1.5.6.tar.gz
cd pip-1.5.6/
/opt/python2.7.8/bin/python setup.py install

# install gevent, pyopenssl, pycrypto
/opt/python2.7.8/bin/pip install gevent
/opt/python2.7.8/bin/pip install pyopenssl
/opt/python2.7.8/bin/pip install pycrypto

# to run goagent, please use:
# /opt/python2.7.8/bin/python where_goagent_proxy_py

#9 Monistra...@gmail.com 
It doesn't affect you when using Goagent. 
That means your certificate database might not be initialized. To fix this, perform:

rm -rf $HOME/.pki
mkdir -p $HOME/.pki/nssdb
certutil -d $HOME/.pki/nssdb -N

Enjoy!
