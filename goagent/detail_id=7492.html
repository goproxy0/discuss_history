Reported by cyrilcui...@gmail.com, Nov 27, 2012 
用电脑访问网页, 提示的错误内容 (有 Google 徽标和坏掉的机器人):

503. That’s an error.
The service you requested is not available at this time.
Service error -27. That’s all we know.

用安卓手机访问网页, 提示证书错误, 不能看到任何 HTML 的内容.

尝试诊断:

0. 流量等各项额度均为 0% .

1. 已按照  Issue 6680  提到的方法更改配置文件. 重新制作并导入了 CA.crt 证书 (包括电脑和手机的, 客户端和系统证书区的). >>可以<<正常访问以下网站 (想说导入证书或重新上传的朋友请绕道): encrypted.google.com, mail.google.com, maps.google.com, code.google.com, ... 无法访问以下网站: wordpress.com, play.google.com.

2. HTTP 网页正常访问或提示连接断开 (The server closed the connection without sending any data). HTTPS 网页出现上述问题.
3. 不同版本结果不同: 1.8.x 和 2.0.x 均有以上错误, 2.1.9-11 则能正常访问. 安卓的应用我用的是老版本的, 所以服务端路径是 /fetch.py 而不是 /2, 也可以访问. 都是 HTTPS 模式.
4. 我同时上传了很多个 app , 于是尝试把不同版本的服务端上传到不同的应用. 经反复替换测试后发现 (由此判断删除再重建 app 不会有效果), 此现象与 app 名称无关, 只受版本号影响.

总之, 各种现象之间互相矛盾, 所以叫诡异的 503. 其实昨天还好好的....
结论: 前阵子 python 引擎遭受不明骚扰, 难道服务器挂了?

用--disable-web-security 参数启动没用, 应该用--ignore-certificate-errors参数启动, 对大部分https 网站都有用, 但对https://twitter.com 无效, 出现的竟然是google报错内容: Google 503. That’s an error. The service you requested is not available at this time. Service error -27. That’s all we know. 原因是twitter.com 被GFW污染了dns, 在hosts文件设正确的twitter.com的ip就行了
