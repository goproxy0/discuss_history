#8 scattere...@gmail.com 
用 gws 或 gvs 作为 gae 可用 ip 判定， 并不准确

以 goagent 默认自带 appid goagent 为例, 访问域名为 goagent.appspot.com
curl -sI https://goagent.appspot.com | grep Server
返回为
Server: Google Frontend

Google Frontend 才应作为gae判定的服务类型

以所谓不支持 gae 的 gvs ip 203.208.41.2 为例
curl -skI https://203.208.41.2 | grep Server
返回 Server: gvs 1.0
但如果通过域名 goagent.appspot.com 以 http 或 https 访问 
(http 访问比用 https 访问开销要小，所耗时间更短)
curl -sI http://goagent.appspot.com --resolve goagent.appspot.com:80:203.208.41.2 | grep Server
curl -skI https://goagent.appspot.com --resolve goagent.appspot.com:443:203.208.41.2 --ciphers ECDHE-RSA-AES128-SHA | grep Server
返回都为 Server: GFE/2.0
并不是 Google Frontend , 所以不支持 gae 

以上 goagent.appspot.com 可以换成更通用的域名 appengine.google.com

用 gws 判定也有类似的问题，记得移动以前就有 ip 支持 gws ， 但实际不可用
