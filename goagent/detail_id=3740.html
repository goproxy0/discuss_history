#4 cyrilcui...@gmail.com 
已尝试找到解决办法.

错误原因可能是GAE修改了对待部署版本的处理方式. 现在新上传的版本不能自动设置为默认版本, 需要手点. 
设置方法: app engine -> your application -> versions -> click the radio box of the newest version -> click "make default" button

注意几点:
1. 如果在上传时, 还没等到提示Checking if deployment succeeded就出现了error 10038, 那么需要关掉上传界面重新运行. 这说明程序没有上传成功. 
2. 多个app需要分开上传, 因为第一个出错以后就会卡住. 
3. 貌似不管是新建app还是延用以前的app, 都会出现这个毛病. 
4. 可以试着用浏览器直接打开yourapp.appspot.com/fetch.py  在make default之前是不能访问的, make之后稍等片刻就提示部署正常了
