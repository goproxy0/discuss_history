可能這個網站故意屏蔽了某些地區的用戶，如果是這樣，GV目前的匿名效果可能不如SS。
如果能繼續開發完善，GV其實是挺有潛力的

把过程稍微总结了一下，供需要的同学参考。如有不对还望指点，我对这些命令一点都不熟悉，都是google来的。

OS：Debian 7.0。选择正确的OS是前提。
VPS：自购搬瓦工
说明：CentOS 6.x默认Python版本为2.6，部署goagent到vps需要2.7版的python。尝试在CentOS 6.0上升级Python版本到2.7，以失败告终，yum在Python2.7下不能使用。换Ubuntu12.04 x86，需要自己先部署Shadowsocks（CentOS不用自己部署SS，搬瓦工有一键部署，这也是我首先尝试在CentOS下部署Goagent的原因，很不幸这个选择最终没有成功）。在部署SS的过程中遇到麻烦，尝试几次后转用Debian 7.0。

I. Install new OS之后首先做一些准备工作。

1. 修改VPS的ssh端口为22。
搬瓦工的ssh端口是随机给的，而goagentvps上传使用的端口默认为22，所以需要手动修改VPS的ssh端口。可以直接在搬瓦工的控制面板里面的File Manager里面，在/etc/ssh/sshd_config中修改最后一行Port为22，保存，然后重启ssh服务（控制面板里的Root shell - basic应该就可以实现）

sudo service ssh restart

2. 把VPS的root密码改成自定义的。搬瓦工会给一个随机的密码，此密码很难记住，对后面使用putty登录很不友好，建议修改成自己能记住的。当然你也可以不改。修改root密码需要首先用搬瓦工给的随机密码登录putty，然后用以下命令修改：

sudo passwd root

重复键入两次新的密码后就修改成功了。

1、2两步之后登录putty就可以用22端口和自定义密码了。


II. 用Debian 7.0系统部署SS。几个命令如下：

1. 安装python-pip、python-m2crypto、supervisor以及shadowsocks
apt-get update
apt-get install python-pip python-m2crypto supervisor
pip install shadowsocks

2. 创建并编辑/etc/shadowsocks.json
第一步安装完shadowsocks并不会在/etc目录下自动生成shadowsocks.json文件，这个文件需要自己创建并写入参数。具体在putty客户端下操作：

vi /etc/shadowsocks.json

然后输入以下内容：
{
"server":"VPS服务器地址",
"server_port":任意填,
"local_port":1080,
"password":"密码",
"timeout":600,
"method":"密码类型：rc4-md5等"
"fast_open": false,
"workers": 1
}

3. 启动shadowsocks服务

ssserver -c /etc/shadowsocks.json

此时可以在PC上打开SS客户端，填入第二步的信息，确定后就可以利用SS翻墙了。但是此时必须保持putty窗口开着，一旦关闭就终止SS服务了。因此需要配置SS在后台运行，以及开机（Debian系统启动）时自动启动SS服务。

4. 创建并编辑/etc/supervisor/conf.d/shadowsocks.conf
同上，/etc/supervisor/conf.d/shadowsocks.conf需要自己手动创建。也是在putty中操作：

vi /etc/supervisor/conf.d/shadowsocks.conf

然后输入以下内容：

[program:shadowsocks]
command=ssserver -c /etc/shadowsocks.json
autorestart=true
user=nobody

如果第2步的local_port < 1024，把上面的 user=nobody 改成 user=root

5. 编辑/etc/default/supervisor文件，在最后加入下面的一行内容： 

vi /etc/default/supervisor

ulimit -n 51200

以上所有涉及vi的操作最后都要保存并退出。命令为：先按ESC退出Insert状态，然后输入":w"，表示写入文件，再输入":q"，退出编辑模式。

6. 启动supervisor服务，监控shadowsocks服务

service supervisor start
supervisorctl reload

此时SS服务就应该在后台运行了。用以下命令可以实现相关功能：

upervisorctl tail -f shadowsocks stderr   //查看shadowsocks日志
supervisorctl restart shadowsocks        //重启shadowsocks进程
supervisorctl start shadowsocks        //启动shadowsocks进程
supervisorctl stop shadowsocks        //停止shadowsocks进程


III. 部署Goagent到VPS。

1. 首先去goagent/server目录上传goagentvps。双击goagentvps.bat
host输入第II/2步里面的VPS服务器地址
username输入root
