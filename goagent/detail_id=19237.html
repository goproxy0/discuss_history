首先，出现这样的提示：
    Permission denied
基本就说明程序需要更高的权限。
    sudo
这个前缀，是「Super User Do」的缩写；换言之，就是以超级用户（*nix 系统是 root 用户）权限执行后续命令。

这里需要 root 权限是因为 dnsforwarder 需要监听 53 端口，普通用户权限是不支持这个操作的，因此必须要 sudo。

昨天给你的建议里，有一条
    sudo mv ./dnsforwarder /usr/bin/dnsforwarder
这条命令的意思是将 dnsforwarder 这个可执行程序放入 /usr/bin/ 这个目录，而 /usr/bin/ 这个目录是所谓 PATH 变量的一部分。这是什么意思呢？PATH 这个变量里，保存了几个路径。你可以用
    echo $PATH
来查看 PATH 里有哪些路径。在终端里执行命令的时候，系统实际上会在 PATH 里记录的这些路径里去寻找相应的可执行程序。比如
    ls
这个命令，就对应着一个可执行程序。你可以用
    which ls
来找到这个程序的具体位置。
    ~$ which ls
    /bin/ls
我们刚才将 dnsforwarder 放入了 PATH 记录的目录里，我们就可以直接在终端里使用
    dnsforwarder
命令来启动程序，而不需要使用完整的路径
    /path/to/dnsforwarder
来执行。

综上，你只需要使用
    sudo dnsforwarder 
就可以启动 dnsforwarder 了。当然，这样子的话，是在前台运行程序——如果你关闭终端的话，dnsforwarder 也会一并关闭。你可以在后面加上 -d 选项，来使 dnsforwarder 在后台运行（像我昨天告诉你的那样）。
    sudo dnsforwarder -d

如果你想了解 dnsforwarder 配置文件的使用方法，可以参看它的官方文档。
https://github.com/holmium/dnsforwarder/wiki/首次使用方法
https://github.com/holmium/dnsforwarder/blob/5/default.config

-----

你的问题更大的可能是 ISP 层面执行了更多的 IP 封锁，这也是为什么 GXW 在不同地区表现不同的原因之一。我建议你尝试 Shadowsocks。
Shadowsocks 有免费的公共服务器，不过速度较慢；你也可以去寻找其他人免费公开的自搭建的服务器，速度较快；或者可以去购买服务器；最一劳永逸的方法是自己搭建一个服务器。
