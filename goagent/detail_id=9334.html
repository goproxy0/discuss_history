talk.google.com 错误解决方法
GAE 只能作为 http (以及伪 https) 代理使用, Chrome 在启动时会用 https 代理去连接 talk.google.com:5222, 但所用的并非是 https 协议的, 走不了 "GAE 代理", 可以通过pac或者浏览器扩展中的代理选择功能让这个请求不走 GAE 从而避免出现错误.
