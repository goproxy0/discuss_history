Issue 6318: 【BUG】2.1.4 与本机ipv6的hosts文件冲突，导致无法打开google.com搜索结果
1 person starred this issue and may be notified of changes. Back to list
Status:  Done
Owner:  ----
Closed:  Dec 2012
Type-Defect
Priority-Medium


 
Reported by hay...@gmail.com, Oct 27, 2012 
Windows 7系统，教育网ipv6，并且在hosts文件内添加了google等服务的IPV6解析。
使用2.1.3、2.1.4版本都发现，goagent与本机的hosts文件冲突。若存在本机google服务的ipv6 hosts解析，则使用goagent会导致google.com的搜索结果打不开，提示错误324：
---------
未收到数据
服务器未发送任何数据，因此无法载入该网页。
以下是一些建议：
请稍后重新加载此网页。
错误 324 (net::ERR_EMPTY_RESPONSE)：服务器已断开连接，且未发送任何数据。
-----------
更换profiles=google_cn\google_hk\google_ipv6均会产生此问题。

如果我在我的hosts文件内把google的ipv6解析删除掉，则一切正常。

注：2.1.0及以前版本均无此问题。我也是在更新2.1.3后才发现。2.1.4也存在此问题
